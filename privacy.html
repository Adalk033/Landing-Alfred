Política de Privacidad

Última actualización: 4 de noviembre de 2025

Alfred es 100% privado. No recopilamos, almacenamos ni compartimos ningún dato personal. Todo se procesa localmente en tu dispositivo. No hay servidores externos, no hay telemetría, no hay seguimiento.

1. Introducción

En Alfred, tu privacidad no es solo una prioridad, es el núcleo de nuestro diseño. Esta Política de Privacidad explica cómo Alfred maneja tus datos y por qué puedes confiar en que nunca dejamos tu dispositivo.

Alfred es un asistente de escritorio con inteligencia artificial completamente local. A diferencia de otros asistentes, no requiere conexión a internet para funcionar y nunca envía tus datos a servidores externos.

2. Información que NO Recopilamos

Alfred NO recopila, transmite ni comparte:

  • Información personal identificable (nombre, email, dirección, etc.)
  • Documentos que procesas con la aplicación
  • Conversaciones con el asistente IA
  • Historial de consultas o búsquedas
  • Datos de uso o telemetría
  • Información del dispositivo o sistema operativo
  • Dirección IP o ubicación geográfica
  • Cookies o identificadores de seguimiento
  • Cualquier otro tipo de dato personal o de uso

3. Cómo Funciona Alfred

Procesamiento 100% Local

Todo el procesamiento de Alfred ocurre exclusivamente en tu dispositivo:

  • IA Local: Utiliza Ollama para ejecutar modelos de lenguaje directamente en tu computadora
  • Base de Datos Local: ChromaDB se ejecuta localmente para búsqueda vectorial
  • Almacenamiento Local: Todos los datos se guardan en tu dispositivo (Windows: AppData, Linux: ~/.alfred, macOS: Application Support)
  • Sin Conexiones Externas: No hay comunicación con servidores externos durante el uso normal

Encriptación de Datos

Tus datos sensibles están protegidos con encriptación AES-256-GCM:

  • Datos personales como RFC, CURP, NSS están encriptados en la base de datos
  • Las claves de encriptación se generan y almacenan únicamente en tu dispositivo
  • Ni siquiera nosotros podemos acceder a tus datos encriptados

4. Almacenamiento de Datos

Alfred almacena datos exclusivamente en tu dispositivo local:

  • Documentos indexados: Fragmentos de texto procesados para búsqueda (embeddings vectoriales)
  • Historial de conversaciones: Tus conversaciones con el asistente, guardadas localmente
  • Caché de respuestas: Respuestas previas para acelerar consultas repetidas
  • Configuración de usuario: Preferencias y ajustes de la aplicación
  • Logs locales: Registros de funcionamiento para diagnóstico (nunca contienen datos personales)

Ubicaciones de almacenamiento:

  • Windows: C:\Users\<Usuario>\AppData\Roaming\Alfred\
  • Linux: ~/.alfred/
  • macOS: ~/Library/Application Support/Alfred/

5. Conexiones a Internet

Alfred requiere conexión a internet ÚNICAMENTE para:

  • Descarga inicial: Descargar modelos de IA desde Ollama (primera vez)
  • Actualizaciones de software: Verificar nuevas versiones (opcional, puedes desactivarlo)

Una vez instalado y con los modelos descargados, Alfred funciona completamente offline. No hay comunicación con servidores durante el uso normal de la aplicación.

6. Código Abierto y Auditable

Alfred es software de código abierto bajo Apache License 2.0:

  • El código fuente está disponible públicamente en GitHub
  • Cualquiera puede auditar el código para verificar nuestras afirmaciones de privacidad
  • La comunidad puede contribuir y mejorar la aplicación
  • No hay código oculto ni funcionalidades no documentadas

7. Terceros y Servicios Externos

Alfred NO utiliza:

  • Servicios de análisis (Google Analytics, etc.)
  • Servicios de publicidad
  • Servicios de seguimiento o rastreo
  • CDNs para contenido dinámico
  • APIs de terceros para funcionalidad principal

Nota sobre Ollama: Ollama es un servidor de modelos de IA que también se ejecuta localmente en tu dispositivo. No envía datos a servidores externos durante el uso normal.

8. Control de tus Datos

Tienes control total sobre tus datos:

  • Eliminar datos: Puedes eliminar conversaciones, documentos indexados o toda la base de datos desde la aplicación
  • Exportar datos: Tus datos están en formatos estándar (SQLite, ChromaDB) que puedes acceder directamente
  • Desinstalar: Al desinstalar Alfred, puedes elegir mantener o eliminar todos los datos locales

9. Seguridad

Medidas de seguridad implementadas:

  • Encriptación AES-256-GCM: Para datos sensibles en la base de datos
  • Context Isolation: Electron ejecuta con aislamiento de contexto activado
  • Sandboxing: Procesos renderer ejecutan en sandbox
  • Sin Node Integration: El renderer no tiene acceso directo al sistema
  • Host fijo: El backend solo escucha en 127.0.0.1 (localhost)

10. Actualizaciones de la Aplicación

Las actualizaciones de Alfred:

  • Se notifican al usuario pero no se instalan automáticamente
  • La verificación de actualizaciones es opcional y puede desactivarse
  • No se recopila información del dispositivo durante la verificación
  • Las actualizaciones se descargan desde repositorios públicos (GitHub Releases)

11. Menores de Edad

Alfred no recopila datos de usuarios de ninguna edad. La aplicación puede ser utilizada por personas de cualquier edad bajo la supervisión de un adulto responsable cuando sea apropiado.

12. Cambios a esta Política

Cualquier cambio a esta Política de Privacidad será:

  • Publicado en esta página con fecha de actualización
  • Notificado a través de la aplicación (para cambios importantes)
  • Disponible en el repositorio de GitHub con historial de cambios

El uso continuado de Alfred después de cambios en la política constituye aceptación de los mismos.

13. Contacto

Si tienes preguntas sobre esta Política de Privacidad:

  • GitHub: Abrir un issue
  • Email: (Disponible en el perfil de GitHub)

Resumen: Alfred está diseñado desde cero para proteger tu privacidad. No recopilamos datos. No compartimos datos. No hay servidores externos. Todo es local. Tu privacidad es absoluta.
